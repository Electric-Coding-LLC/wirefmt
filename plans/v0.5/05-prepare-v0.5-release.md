# Prepare v0.5 release

[Back to Execution Map](./EXECMAP.md)

## Goal

Update the release-facing docs and verification notes so `v0.5` is ready for a
manual release flow after the bounded small-gap adjacent-box work is complete.

## Tasks

- Update release notes and docs to describe the supported one- to three-space
  sibling-box frame and the nearby shapes that remain unsupported.
- Confirm the standard local gate still covers the changed scope.
- Keep the release message centered on bounded small-gap support, not general
  layout intelligence.

## Constraints

- Do not run release, publish, or tag commands as part of this step.
- Keep the release narrative consistent with the actual shipped contract and
  test coverage.

## Exit Criteria

- Docs and release notes describe the `v0.5` scope accurately.
- The repo is left ready for the normal manual release flow, without claiming
  the release has already happened.
